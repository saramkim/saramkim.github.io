프리코스

(2022/10/26 ~ 2022/11/22) 4주간 우테코 5기 프리코스에 참여하였는데, 정말 뜻깊고 많은 것을 배운 한 달이었다.

합격하여 본 코스에도 참여하였으면 더 훌륭한 신입 개발자가 될 수 있었을텐데 아쉽다.

이전에 몇 회사에 지원하여 탈락의 고배를 경험하였지만, 어느 이유에서 탈락하였는지 피드백이 없으니 보충해야 할 부분을 알 수 없었다.

그런 나에게 우테코 프리코스는 학습의 방향성과 나의 부족한 부분을 짚어주는 친절한 가이드가 되어 주었다.

기능 목록

미션 구현을 위해 필요한 기능, 즉 구현할 기능의 목록을 먼저 작성한 후에 코딩하여 기능 단위로 커밋한다.

  • 깃 커밋 메세지 컨벤션을 준수하며 커밋 메세지를 의미 있게 작성한다.

  • 예외적인 상황도 기능 목록에 작성한다.

  • 기능 목록을 지속해서 업데이트한다.


네이밍

이름을 통해 의도를 드러낸다. 변수명, 함수명, 클래스명을 지을 때 역할에 맞게 적절한 이름을 부여한다.

  • 축약하지 않는다.

  • 값을 하드 코딩하지 않고 상수를 만든다.

  • 컨벤션, 규칙에 맞게 네이밍하면 좋다.


함수 분리

한 함수가 한 가지 기능만 담당하게 한다. 최대한 함수를 분리한다.

  • 최대 10라인, 최대 두 번 들여쓰기, 최대 파라미터 3개와 같이 함수를 제한하면 함수 분리에 도움이 된다.

  • 단위 테스트를 작성하면 함수 분리의 필요성을 알 수 있다.


단위 테스트

작은 기능부터 테스트를 작성하여 도메인 로직에 단위 테스트를 작성한다.

  • 테스트의 중요한 목적 중 하나는 내가 작성하는 코드에 대해 빠르게 피드백을 받는 것이다. 리펙터링에 도움된다.

  • 도메인(비즈니스) 로직과 UI 로직을 구분한다.

  • 예외 상황을 고려해 프로그래밍하는 습관을 들인다.


객체지향

프로그래밍 요구사항이 전반적으로 객체지향 패러다임을 기반으로 한다.

따라서 자연스럽게 객체지향적 사고를 할 수 있게 하고, 객체지향의 필요성을 느낄 수 있다.

또한 올바른 객체지향의 접근법을 배울 수 있었다.


의식적인 연습

프로그래밍 요구사항을 통해 목적의식 있는 연습을 할 수 있게 설계되어 있었다.

매주 요구사항이 늘어나고 더욱 제한하여 항상 의식적으로 프로그래밍할 수 있었고, 그에 따라 효과적으로 학습하여 매주 성장할 수 있었다.

기능 목록을 작성하여 구체적인 목표가 있으니 더욱 효과적이었다.
